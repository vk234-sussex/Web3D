My 3D App Deeper Understanding

You must list here your Deeper Understanding components of your 3D App

List here those components of your 3D App module where you feel you should score 'Deeper Understanding' marks according to the specified rubric. Some examples are given, you may list more than 4, but a maximum 4 x 5 Marks will be awarded.

  • Rendering is done making use of Unity compiled to WebAssembly for rendering of 3D models, with models loaded asynchronously

  • Bound JavaScript to Unity C# for calling of events, and abstraction of logic in the renderer (which is transpiled to C++, and then compiled with Unity's IL2CPP down to WebAssembly, such that it can be used in the browser)

  • Modelling completed using Blender instead of 3DS Max

  • Utilised Adobe Substance 3D Painter to add normal mapping details to objects, as well as completing high quality renders

  • Built custom templating system in JavaScript, which allows for page components to be easily re-used and parameterised, with this data stored on the server

  • Developed custom shader for the Smartwater bottle, such that the filled "water" on the side of the bottle moves up and down along a sine wave, forming ripples in the water

  • Made use of Bootstrap 5 instead of Bootstrap 4 that is utilised in the labs

  • APIs and tools built, creating an addon for the Unity Editor to reduce iteration time, with compiling the changes to an asset bundle, and then deploying this bundle to the server being done in two clicks

  • Lerping function used to rotate objects back to the expected rotation smoothly, accounting for users potentially rotating inline with the expected movement - such that objects can be "spun" in the 3D viewport
